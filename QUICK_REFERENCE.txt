🔑 AUTHENTICATION
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

Token-Based Auth Flow:
  1. User POSTs to /api/auth/signup/ with email/password
  2. Server returns auth token
  3. User includes token in future requests: "Authorization: Token {token}"
  4. All /predict/, /skill-gap/, /bookmarks/ endpoints require token

Example Request:
  curl http://localhost:8000/api/predict/ \
    -H "Authorization: Token abc123def456..." \
    -H "Content-Type: application/json" \
    -d '{"answers":{...}}'

  → Make sure you're in backend/ directory
  → Verify 'career_app' is in INSTALLED_APPS in settings.py

OperationalError: no such table
  → Run: python manage.py migrate

Token not found on protected endpoints
  → Use Authorization header: "Authorization: Token {your_token}"
  → Get token from signup/login response

Port 8000 already in use
  → Use: python manage.py runserver 8001
  → Or kill process: lsof -ti:8000 | xargs kill -9
